Relative Font size

Relative length units specify a length relative to another length property.

And Yes, using a relative size such as %, em or rem matters on styling size

of an element. because first of all those relative size units are scalable font size units as a result it is related to the size of the container or related to the root element (which is Html in this case) instead of the parent elemnet. second relative size units are more flexible when manupulating webpages. besides relative size units are more clear and are practical in creating perfecly scalable layout. for example:
Em --he “em” is a scalable unit that is used in web document media. An em is equal to the current font-size, for instance, if the font-size of the document is 12pt, 1em is equal to 12pt. Ems are scalable in nature, so 2em would equal 24pt, .5em would equal 6pt, etc. Em are becoming increasingly popular in web documents due to scalability and their mobile-device-friendly nature. While em is relative to the font-size of its direct or nearest parent, rem is only relative to the html (root) font-size. em gives the ability to control an area of a design. As in, scale the type in that specific area relatively. rem gives the ability to scale type across the entire page easily.

Default Browser Width

Setting the font size in pixel values (px) is a good choice when you need pixel accuracy. A px value is static. This is an OS-independent and cross-browser way of literally telling the browsers to render the letters at exactly the number of pixels in height that you specified. The results may vary slightly across browsers, as they may use different algorithms to achieve a similar effect. Font sizing settings can also be used in combination. For example, if a parent element is set to 16px and its child element is set to larger, the child element displays larger than the parent element in the page

Css3 Changes

CSS3 is th latest version of CSS(Cascading Style Sheets) languages. It extends CSS2.1 and has a lot of new features added to it.some of the major changes are: *Animations and transitions there There are two ways to create CSS animations. The first is very easy, it is done through animating the changes of CSS properties with the transition declaration. With transitions, you can create hover or mouse down effects, or you can trigger the animation by changing the style of an element with JavaScript. You can see the transition below by hovering over the planet - this will cause the rocket to close in. The second way for defining animations is a bit more complicated - it involves the description of specific moments of the animation with the code>@keyframe rule. This allows you to have repeating animations that don't depend on user actions or JavaScript to get triggered. Hit the Edit button to see the code. *Advanced Selectors by assigning Id to the elements we are made able to use different powerful selectors *Rounded Corners Rounded corner elements can spruce up a website, but creating a rounded corner requires a designer to write a lot of code. Adjusting the height, width and positioning of these elements is a never-ending chore because any change in content can break them. *Box Shadow A box shadow allows you to create a drop shadow for an element. Usually this effect is achieved using a repeated image around the element. However, with the property box-shadow this can be achieved by writing a single line of CSS code. *Text Shadow The new text-shadow property allows you to add drop shadows to the text on a webpage. Prior to CSS 3, this would be done by either using an image or duplicating a text element and then positioning it. A similar property called box-shadow is also available in CSS 3. After previously removing this property from the CSS 3 Backgrounds and Borders Module, the W3C added it back in the last working draft. CSS 3 addresses this problem by introducing the border-radius property, which gives you the same rounded-corner effect and you don't have to write all the code. Here are examples for displaying rounded corners in different places of a website. *Css Rgba Opacity The RGB property in CSS is used to set colors for different elements. CSS 3 introduces a modification and added opacity manipulation to this property. RGB has been transformed to RGBA (Red Green Blue Alpha channels), which simplifies how you control the opacity of elements.
